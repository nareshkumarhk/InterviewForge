Interviewer: Hi, thanks for joining. Let's start with some fundamentals. Can you explain how HashMap works in Java?

Candidate: Sure. HashMap uses hashing to store key-value pairs. It calls the hashCode() method on the key to determine which bucket to place the entry in. When there's a collision - when two keys hash to the same bucket - it uses chaining, which is a linked list of entries in that bucket. In Java 8 and later, if a bucket accumulates more than 8 entries, it converts the linked list to a red-black tree for better lookup performance.

Interviewer: Good. What happens when the HashMap needs to resize?

Candidate: When the number of entries exceeds the load factor times capacity - the default load factor is 0.75 - it doubles the capacity and rehashes all existing entries into the new buckets. This is an O(n) operation.

Interviewer: Let's do a coding problem. Given a string, find the length of the longest substring without repeating characters.

Candidate: Okay, let me think. A brute force approach would be to check every possible substring and see which one has no repeating characters. That would be O(n³) or O(n²) time which isn't great.

A better approach is to use a sliding window with a hash set. I'll maintain a window with two pointers - left and right. I expand right, and if I encounter a duplicate character, I move left forward until the duplicate is gone. At each step I track the maximum window size.

Interviewer: Can you code that up?

Candidate: Sure.

def length_of_longest_substring(s):
    char_set = set()
    left = 0
    max_length = 0

    for right in range(len(s)):
        while s[right] in char_set:
            char_set.remove(s[left])
            left += 1
        char_set.add(s[right])
        max_length = max(max_length, right - left + 1)

    return max_length

Interviewer: What's the time and space complexity?

Candidate: Time complexity is O(n) since each character is added and removed at most once. Space complexity is O(min(m, n)) where m is the size of the character set and n is the string length.

Interviewer: What edge cases should we handle?

Candidate: Empty string - would return 0, which is correct since left starts at 0 and max_length stays 0. Single character - works fine. String with all same characters like "aaaa" - the window would shrink to size 1 each time. All unique characters - no shrinking needed, returns full length.
